Detailed Description

template<size_t kMaxConnectionCount, Time::Source kTimeSource = Time::Source::kSystem>
class chip::Transport::PeerConnections< kMaxConnectionCount, kTimeSource >

Handles a set of peer connection states.

Intended for:

  • handle connection active time and expiration
  • allocate and free space for connection states.

Allocates a new peer connection state state object out of the internal resource pool.

Parameters

peerKeyIdrepresents the encryption key ID assigned by peer node
localKeyIdrepresents the encryption key ID assigned by local node
state[out] will contain the connection state if one was available. May be null if no return value is desired.
Note
the newly created state will have an 'active' time set based on the current time source.
Returns
CHIP_NO_ERROR if state could be initialized. May fail if maximum connection count has been reached (with CHIP_ERROR_NO_MEMORY).
