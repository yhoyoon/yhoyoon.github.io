Detailed Description

Adaptation for LwIP ip4_addr_t type.

File contains definitions on how a connection to a peer can be defined.

Defines state relevant for an active connection to a peer.

defines a generic time source interface that uses a real clock at runtime but can be substituted by a test one for unit tests.

chip-specific nlassert.h Overrides

Before LwIP 2.0.0, the ip_addr_t type alias referred to a structure comprising an IPv4 address. At LwIP 2.0.0 and thereafter, this type alias is renamed ip4_addr_t and ip_addr_t is replaced with an alias to a union of both. Here, the ip4_addr_t type alias is provided even when the LwIP version is earlier than 2.0.0 so as to prepare for the import of the new logic.

◆ CastToSigned()

template<typename T >
std::enable_if<std::is_unsigned<T>::value, typename std::make_signed<T>::type>::type chip::CastToSigned ( T  arg)

A function to reverse the effects of a signed-to-unsigned integer cast.

If the argument is small enough to be representable as a positive signed integer, returns that integer. Otherwise, returns a negative integer which would, if cast to the type of the argument, produce the given value.

So for example, if a uint8_t with value 254 is passed in this function will return an int8_t with value -2.

Note
This function might become unnecessary if C++20 standardizes 2s-complement signed integers and defines casting of out-of-range values to signed types.
