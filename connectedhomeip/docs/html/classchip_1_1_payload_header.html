Detailed Description

Handles encoding/decoding of CHIP payload headers.

Payload headers are ENCRYPTED and are placed at the start of an encrypted message payload.

Set the message type for this header.

This requires setting the protocol id as well, because the meaning of a message type is only relevant given a specific protocol.

This should only be used for cases when we don't have a strongly typed message type and hence can't automatically determine the protocol from the message type.
